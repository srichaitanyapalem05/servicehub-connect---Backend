Build a complete, production-ready backend for a **Multi-Vendor Service Booking Platform** using **Node.js, Express.js, and MongoDB (Mongoose ORM)** following clean architecture and best practices.

---

## 🧱 Tech Stack Requirements

* Node.js + Express.js
* MongoDB with Mongoose
* JWT Authentication
* bcrypt for password hashing
* dotenv for environment variables
* express-validator for validation
* multer (optional for file uploads like vendor/service images)
* nodemon for development
* MVC folder structure
* RESTful API design

---

## 📁 Folder Structure

Create a scalable architecture:

* /config (DB connection, env setup)
* /controllers (business logic)
* /models (Mongoose schemas)
* /routes (API routes)
* /middlewares (auth, error handling, role-based access)
* /utils (helpers, constants)
* /services (optional business layer)
* server.js (entry point)

---

## 🔐 Authentication & Authorization

Implement secure JWT-based authentication:

* User roles:

  * customer
  * vendor
  * admin

* Features:

  * Register (User & Vendor)
  * Login
  * Password hashing using bcrypt
  * JWT token generation
  * Middleware to protect routes
  * Role-based access control (RBAC)

---

## 👤 User Module APIs

* POST /api/auth/register (customer)
* POST /api/auth/login
* GET /api/users/profile (protected)
* GET /api/services (browse services)
* GET /api/services/:id (service details)
* POST /api/bookings (create booking)
* GET /api/bookings/my (user booking history)
* POST /api/reviews (add review)

Features:

* Search + filter services (category, price, rating)
* Booking history with status

---

## 🧑‍🔧 Vendor Module APIs

* POST /api/vendor/register
* POST /api/vendor/login
* GET /api/vendor/dashboard
* POST /api/services (add service)
* PUT /api/services/:id (update service)
* DELETE /api/services/:id (delete service)
* GET /api/vendor/bookings
* PATCH /api/vendor/bookings/:id (update status)

Features:

* Vendors can manage their services
* Accept/reject bookings
* View earnings (optional)

---

## 🛡️ Admin Module APIs

* GET /api/admin/users
* GET /api/admin/vendors
* PATCH /api/admin/vendors/:id/approve
* DELETE /api/admin/users/:id
* GET /api/admin/bookings
* GET /api/admin/reports

Features:

* Approve/reject vendors
* Monitor platform activity
* Analytics (basic counts)
